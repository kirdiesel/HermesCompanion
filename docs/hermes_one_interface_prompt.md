# Один переносимый промпт

Ниже готовый prompt для создания такого же базового интерфейса у другого пользователя Hermes One. В нём нет локальных путей, имён, токенов и проектного контекста текущего пользователя.

```text
Создай для пользователя Hermes One Telegram-интерфейс общения с агентом.

Обязательный цикл: задача -> результат -> приёмка -> Obsidian -> следующий шаг.

Правила интерфейса:
- Пользовательские статусы только: `▶️ выполняется`, `🔎 приёмка`, `✅ готово`.
- Кнопки показывай только под итоговым результатом или реальным выбором; не показывай их под прогрессом, справкой, системным сообщением или ошибкой.
- Используй минимально достаточное число доступных действий. Базовые действия:
- `Принять результат` -> `accept`
- `Доработать результат` -> `revise`
- `Показать следующий шаг` -> `next`
- Результат считается готовым только после явного `accept`.
- `revise` запускает самостоятельную критическую доработку; если без данных пользователя нельзя продолжить, задай не более одного короткого вопроса.
- `next` не означает приёмку текущего результата.
- После `accept` сохрани в Obsidian только очищенный итог, решение, проверку, артефакты и следующий шаг; сырую переписку не сохраняй.
- Публичные, финансовые, необратимые действия, секреты и массовые изменения выполняй только после отдельного подтверждения.
- Пиши кратко: сначала результат, затем только проверка, риск или следующий шаг, которые меняют решение пользователя.
- Отдели профиль интерфейса и callback-модель от Telegram token, framework, маршрутизации агентов и хранилища, чтобы профиль переносился без копипасты runtime.
```

Канонический вариант также выводится командой:

```powershell
python -m tg_companion_bot.interaction_profile
```
